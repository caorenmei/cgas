# MiniGas V2

## 概述

MiniGas V2 是 MiniGas 的第二个版本，是完全重构的版本，主要目标用于在服务器计算实体之间的相互作用，且方便与现有系统集成。

## 数据结构

### 通用定义

```lua

---@alias mini_gas.ID integer | string
---@alias mini_gas.Tag string

---@enum mini_gas.EModifierOp
---@field Add 加法
---@field Multiply 乘法
---@field Override 覆盖

---能力激活策略，只支持一种: Passive（被动），表示能力在满足条件时自动激活，无需玩家主动触发。
---@enum mini_gas.EAbilityActivationPolicy
---@field Passive 被动

---@class mini_gas.AttributeDef
---@field id mini_gas.ID
---@field min? number
---@field max? number
---@field default? number

---@class mini_gas.Defs
---@field attribute_defs table<mini_gas.ID, mini_gas.AttributeDef>
---@field effect_defs table<mini_gas.ID, mini_gas.EffectDef>
---@field ability_defs table<mini_gas.ID, mini_gas.AbilityDef>

```

### 系统上下文

```lua

---系统状态接口，由库的使用者实现
---@class mini_gas.IContext

```

### 实体定义

IEntityState 定义了实体的状态，实体的访问，操作由 IEntityModule 提供，IEntityModule 有如下字段：
- 静态标签：实体固有的标签，不会随时间变化。
- 属性：实体的数值属性，如生命值、魔法值等。
- 静态技能：实体固有的技能，不会随时间变化。

```lua

---实体状态
---@class mini_gas.IEntityState

---实体模块接口，提供访问实体状态的函数
---@class mini_gas.IEntityModule
---@field static_tags fun(entity: mini_gas.IEntityState): (fun(entity: mini_gas.IEntityState, s: mini_gas.Tag): mini_gas.Tag), IEntityState 实体的静态标签迭代器
---@field static_tags_size fun(entity: mini_gas.IEntityState): integer 实体的静态标签数量
---@field has_static_tag fun(entity: mini_gas.IEntityState, tag: mini_gas.Tag): boolean 实体是否具有某个静态标签
---@field attributes fun(entity: mini_gas.IEntityState): (fun(entity: mini_gas.IEntityState, id: mini_gas.ID): mini_gas.ID, number), IEntityState 实体的属性迭代器
---@field attributes_size fun(entity: mini_gas.IEntityState): integer 实体的属性数量
---@field has_attribute fun(entity: mini_gas.IEntityState, id: mini_gas.ID): boolean 实体是否具有某个属性
---@field get_attribute fun(entity: mini_gas.IEntityState, id: mini_gas.ID): number 实体的属性值
---@field static_abilities fun(entity: mini_gas.IEntityState): (fun(entity: mini_gas.IEntityState, def_id?: mini_gas.ID): mini_gas.ID), IEntityState 实体的静态技能迭代器
---@field static_abilities_size fun(entity: mini_gas.IEntityState): integer 实体的静态技能数量
---@field has_static_ability fun(entity: mini_gas.IEntityState, def_id: mini_gas.ID): boolean 实体是否具有某个静态技能

```

### 世界状态

IWorldState 定义了世界的状态，世界的访问，操作由 IWorldModule 提供，IWorldModule 功能如下：
- 实体访问：提供访问世界中实体的函数，包括实体的迭代器、数量、是否存在某个实体、获取某个实体的状态等。
- 求值函数：提供一个求值函数，计算实体系统对世界状态的影响。

```lua

--- 世界状态接口
---@class mini_gas.IWorldState

---@class mini_gas.IEvaluation
---@field grant_tags fun(entity: mini_gas.IEntityState, tags: mini_gas.Tag[], ...) 对实体应用标签，可能多次调用添加多个标签
---@field apply_attribute fun(entity: mini_gas.IEntityState, id: mini_gas.ID, value: number, ...) 对实体应用属性变化，可能多次调用应用多个属性变化

---@
---世界模块接口，提供访问世界状态的函数
---@class mini_gas.IWorldModule
---@field entities fun(context: mini_gas.IContext, world: mini_gas.IWorldState): (fun(world: mini_gas.IWorldState, id?: mini_gas.ID): mini_gas.ID, mini_gas.IEntityState, mini_gas.IEntityModule), mini_gas.IWorldState 世界中实体的迭代器
---@field entities_size fun(context: mini_gas.IContext, world: mini_gas.IWorldState): integer 世界中实体的数量
---@field has_entity fun(context: mini_gas.IContext, world: mini_gas.IWorldState, id: mini_gas.ID): boolean 世界中是否存在某个实体
---@field get_entity fun(context: mini_gas.IContext, world: mini_gas.IWorldState, id: mini_gas.ID): mini_gas.IEntityState, mini_gas.IEntityModule 获取世界中某个实体的状态
---@field evaluate fun(context: mini_gas.IContext, world: mini_gas.IWorldState, evaluation: mini_gas.IEvaluation, ...): void 计算实体系统对世界状态的影响

```

### ModifierDef

ModifierDef 定义了 Modifier 的属性和行为，定义如下：

```lua

---@alias mini_gas.ModifierAttributeEval fun(context:mini_gas.IContext, world_state: mini_gas.IWorldState, entity: mini_gas.IEntityState, def: mini_gas.ModifierDef, id?: mini_gas.ID, value?: number): [mini_gas.ID, number], mini_gas.ModifierAttributeEval?

---@class mini_gas.ModifierDef
---@field attribute [mini_gas.ID, number] | mini_gas.ModifierAttributeEval
---@field op mini_gas.EModifierOp
---@field allof_tags? mini_gas.Tag[]
---@field anyof_tags? mini_gas.Tag[]
---@field noneof_tags? mini_gas.Tag[]

```

> attribute 字段可以是一个固定的属性 ID 和数值，也可以是一个函数，如果是函数，则返回一个属性 ID 和数值的元组以及一个可选的函数，如果返回了可选的函数，则递归调用该函数。

### EffectDef

EffectDef 定义了 Effect 的属性和行为，定义如下：

```lua

---@class mini_gas.EffectDef
---@field id mini_gas.ID
---@field modifiers mini_gas.ModifierDef[]
---@field grant_tags? mini_gas.Tag[]
---@field allof_tags? mini_gas.Tag[]
---@field anyof_tags? mini_gas.Tag[]
---@field noneof_tags? mini_gas.Tag[]

```

### AbilityDef

AbilityDef 定义了 Ability 的属性和行为，定义如下：

```lua

---@class mini_gas.AbilityDef
---@field id mini_gas.ID
---@field activation_policy mini_gas.EAbilityActivationPolicy
---@field effects mini_gas.ID[]
---@field can_activate fun(context:mini_gas.IContext, defs: mini_gas.Defs, world_state: mini_gas.IWorldState, entity: mini_gas.IEntityState, def: mini_gas.AbilityDef): boolean

```

> 由于 Ability 都是静态定义的，所有不提供 grant_tags 和标签要求字段。

## 系统流程

系统的核心流程如下：
1. 定义配置：开发者定义 AttributeDef、ModifierDef、EffectDef 和 AbilityDef，构建 Defs。
2. 定义实体状态和实体模块：开发者实现 IEntityState 和 IEntityModule，定义实体的状态和访问方式。
3. 定义世界状态和世界模块：开发者实现 IWorldState 和 IWorldModule，定义世界的状态和访问方式。
4. 调用 IWorldModule.evaluate：计算实体系统对世界状态的影响。
